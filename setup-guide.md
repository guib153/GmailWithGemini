# ✉️ GmailWithGemini 智慧郵件分類系統 — 免費版設定教學 (v3.0)

---

## 📋 前言

**GmailWithGemini** 是一套基於 **Google Gemini AI** 的智慧郵件分類系統。它能自動讀取你 Gmail 收件匣中的未讀郵件，透過 AI 分析郵件內容，並自動套用對應的 Gmail 標籤，讓你的信箱井然有序。

### ✨ 核心特色 (v3.0 全新升級)

- 🤖 **AI 智慧分類與學習** — 使用 Gemini AI 分析郵件，並會「自動記憶」寄件人特徵，未來同一人來信秒速分類不需再問 AI！
- 🎛️ **完全客製化 Prompt** — 內建 `AI_PromptConfig` 面板，讓你隨時在試算表上自由新增分類、修改判斷條件與提供學習範例。
- 📦 **API 批次處理** — 突破傳統限制，每次傳送 10 封信件給 AI 一次判斷，大幅提升速度並將 API 消耗降低 90%。
- 💰 **完全免費** — 使用未啟用 Billing 的 GCP 專案，配合批次處理，不用花任何一毛錢。
- ⏰ **全自動執行與晚報** — 每 1 小時自動執行一次，每天晚上 20:00 自動寄送專屬「每日整理摘要」至信箱。
- 🔍 **未分類覆核機制** — 針對 AI 無法判定的疑難雜症，自動移入 `AI_Uncategorized` 區，讓你手動覆核並教導 AI。

## 🛠️ 第一步：取得免費的 Gemini API 金鑰

> [!CAUTION]
> **這是最關鍵的步驟！** 你必須使用「未啟用 Billing」的 GCP 專案，否則會被收費。

### 操作步驟

1. 開啟 [Google AI Studio](https://aistudio.google.com/)
2. 使用你的 **Google 帳號**登入
3. 點擊左側選單中的 **「Get API key」**
4. **⚠️ 重要：選擇「Create API key in new project」**
   - 這會自動建立一個**全新的、未啟用 Billing 的 GCP 專案**
   - 這是確保免費使用的關鍵！
5. 複製產生的 API Key（格式如 `AIzaSy...`），稍後會用到

> [!WARNING]
> **千萬不要選擇已啟用 Billing 的現有專案！** 如果你的 GCP 專案已綁定信用卡，所有 API 呼叫都會被收費。請務必使用「Create API key in new project」來建立全新專案。

## 💻 第二步：建立 Google Apps Script 專案

### 建立專案與貼上程式碼

1. 前往 [Google Apps Script](https://script.google.com/)
2. 點擊左上角的 **「新增專案」**
3. 點擊左上角的「未命名的專案」，重新命名為 **`GmailWithGemini`**
4. 在編輯器中，刪除預設的 `Code.gs` 所有內容
5. 開啟本專案提供的 `Gmail-with-gemini.gs` 檔案，**複製全部內容**，貼到 `Code.gs` 中
6. 點擊編輯器上方的 **「+」** 按鈕 → 選擇 **「指令碼」**，建立 **`CHANGELOG`**
7. 複製本專案 `CHANGELOG.gs` 的內容貼入其中
8. 點擊 **💾 儲存**（或按 `Ctrl + S`）

### 設定 appsscript.json 資訊清單

> [!IMPORTANT]
> 這個步驟非常重要！`appsscript.json` 定義了腳本所需的 Gmail、Spreadsheet 以及發信權限。

1. 點擊左側的 **「專案設定」⚙️**（齒輪圖示）
2. **勾選**「在編輯器中顯示『appsscript.json』資訊清單檔案」
3. 回到 **「編輯器」**（左側 `< >` 圖示），點擊新出現的 `appsscript.json`
4. **刪除所有內容**，將本專案提供的 `appsscript.json` 內容**完整覆蓋貼入**
5. 點擊 **💾 儲存**

---

## 🔑 第三步：設定 API 金鑰

> [!NOTE]
> 將 API Key 存放在「指令碼屬性」中，而非直接寫在程式碼裡，這是安全最佳做法。

1. 點擊左側的 **「專案設定」⚙️**
2. 向下滑動到 **「指令碼屬性」** 區塊，點擊 **「新增指令碼屬性」**
3. 填入：
   - **屬性（Property）**：`GEMINI_API_KEY`
   - **值（Value）**：貼上第一步取得的 API Key
4. 點擊 **「儲存指令碼屬性」**

---

## 🔍 第四步：驗證 API Key 狀態

1. 回到 **「編輯器」**
2. 在上方的**函式下拉選單**中，選擇 **`checkApiKeyStatus`**
3. 點擊 **「執行」▶️**
4. **首次執行時，系統會要求授權：**
   - 點擊 **「審查權限」** → 選擇你的 Google 帳號
   - 如果出現警告，點擊 **「進階」** → **「前往 GmailWithGemini（不安全）」** → **「允許」**
5. 查看下方的 **「執行記錄」**，確認顯示 API Key 有效的訊息

---

## 🚀 第五步：手動測試與初次建立工作表

確認 API Key 有效後，進行第一次分類測試！

1. 在函式下拉選單中選擇 **`autoOrganizeGmailWithGemini`**
2. 點擊 **「執行」▶️**
3. 首次執行時，系統會在你的 Google 雲端硬碟建立一個名為 **`GmailWithGemini_Rules`** 的試算表。這包含了所有的動態配置與學習資料庫。
4. 等待執行完成後，請打開該試算表，你將會看到以下重要分頁：
   - `AI_PromptConfig`：**【重要】在這裡自訂你的專屬分類與學習範例！**
   - `AI_LearningRules`：AI 自動記下的寄件人分類規則。
   - `AI_Uncategorized`：AI 判斷失敗或你指定為「未分類」的信件會出現在這，你可以在此人工補上分類。
   - `AI_Execution_Log`：每次整理的結果統計。

## ⏰ 第六步：一鍵啟動「全自動排程」

不要再手動點擊了！讓機器人為你代勞：

1. 在函式下拉選單中選擇 **`setupTriggers`**
2. 點擊 **「執行」▶️**
3. ✅ 完成！系統會自動幫你建立好所有排程：
   - **每 1 小時**：自動醒來讀取並分類新信件。
   - **每天 20:00**：自動寄送一封「每日 AI 分類晚報」到你的信箱，報告今天總共整理了多少信件。

## ⚙️ 進階教學：如何教導 AI 與新增分類？ (v3.0)

在 v3.0 版本中，腳本再也不限制你要分幾類，一切由你決定！

1. 打開雲端硬碟中的 **`GmailWithGemini_Rules`** 試算表
2. 切換至 **`AI_PromptConfig`** 分頁
3. **新增/修改類別**：在「AI 分類與描述設定區」，直接打上你想要的類別名稱與詳細描述。AI 在下一次執行時，就會完全聽從你的新設定來分類！
4. **提供範例**：在「Few-Shot 學習範例區」，你可以寫下你希望 AI 怎麼處理特定信件。例如：「如果看到 Uber Eats，就把分類指定為『個人消費』，緊急度設為『低』」。

> [!TIP]
> 如果發現 AI 把某封信分錯了，你可以把那封信的發件人與標題加進 **Few-Shot 學習範例區**，或者在 **`AI_Uncategorized`** 手工輸入並指定給它，下次 AI 就不會再犯同樣的錯囉！

## ❓ 常見問題

### Q: 什麼是批次處理？免費額度夠用嗎？
v3.0 導入了批次處理架構。過去整理 50 封信需要呼叫 50 次 API；現在程式會將 10 封信打包成一包傳給 AI，**處理 50 封信只需 5 次 API 呼叫**！配合每天 1,500 次的免費額度，現在你一天處理破千封信件都輕而易舉！

### Q: 為什麼有些信一直被貼上「未分類」？
當系統遇到難以判斷、API 解析失敗或你不小心把緊急度格式填錯時，為了保護整個程式不卡死，系統會將信件貼上 `AI/未分類` 標籤，並記錄到 `AI_Uncategorized` 工作表。請去試算表幫它人工歸類即可！

### Q: 出現 429 錯誤或 Gmail API 403 錯誤怎麼辦？
- **429 錯誤**：代表請求過於頻繁。腳本已內建「智慧指數退避機制」，會自動等待並重試，無需理會。
- **403 Gmail API disabled**：這只會影響「自動把信件移入 Gmail 原生『促銷內容』分頁」的功能，並不影響標籤。如果你想要這功能，請在編輯器左側的「服務(Services)」中點擊「+」，新增「Gmail API」即可。

### Q: 如何停止自動執行？
在編輯器中選擇 `removeTriggers` 函式，點擊「執行」▶️ 即可一鍵清除所有定時排程。
