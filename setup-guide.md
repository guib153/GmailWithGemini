# ✉️ GmailWithGemini 智慧郵件分類系統 — 免費版設定教學 (v2.0)

---

## 📋 前言

**GmailWithGemini** 是一套基於 **Google Gemini AI** 的智慧郵件分類系統。它能自動讀取你 Gmail 收件匣中的未讀郵件，透過 AI 分析郵件內容，並自動套用對應的 Gmail 標籤，讓你的信箱井然有序。

### ✨ 核心特色

- 🤖 **AI 智慧分類** — 使用 Google Gemini AI 自動分析郵件並歸類
- 💰 **完全免費** — 使用未啟用 Billing 的 GCP 專案，不需要花任何一毛錢
- ⏰ **全自動執行** — 每 6 小時自動執行一次，不需要開電腦
- 📊 **完整日誌** — 自動記錄每封信的分類結果與執行統計
- 🔄 **內建重試機制** — 遇到 API 錯誤會自動重試，無需人工介入

---

## 🛠️ 第一步：取得免費的 Gemini API 金鑰

> [!CAUTION]
> **這是最關鍵的步驟！** 你必須使用「未啟用 Billing」的 GCP 專案，否則會被收費。

### 操作步驟

1. 開啟 [Google AI Studio](https://aistudio.google.com/)
2. 使用你的 **Google 帳號**登入
3. 點擊左側選單中的 **「Get API key」**
4. **⚠️ 重要：選擇「Create API key in new project」**
   - 這會自動建立一個**全新的、未啟用 Billing 的 GCP 專案**
   - 這是確保免費使用的關鍵！
5. 複製產生的 API Key（格式如 `AIzaSy...`），稍後會用到

> [!WARNING]
> **千萬不要選擇已啟用 Billing 的現有專案！** 如果你的 GCP 專案已綁定信用卡，所有 API 呼叫都會被收費。請務必使用「Create API key in new project」來建立全新專案。

### 如何確認專案未啟用 Billing？

1. 前往 [GCP Console — 帳單管理](https://console.cloud.google.com/billing)
2. 在左上角的專案選擇器中，選擇你剛才建立 API Key 的專案
3. 檢查帳單狀態：
   - ✅ 如果顯示 **「此專案沒有帳單帳戶」** → 正確！你可以免費使用！
   - ❌ 如果顯示**已連結帳單帳戶** → 這個專案的 API 呼叫會收費，請勿使用此專案的 API Key

> [!TIP]
> 建議將 API Key 先暫時貼到記事本中保存，避免遺失。後續步驟會教你安全地存放到 Apps Script 的指令碼屬性中。

## 💻 第二步：建立 Google Apps Script 專案

### 建立專案與貼上程式碼

1. 前往 [Google Apps Script](https://script.google.com/)
2. 點擊左上角的 **「新增專案」**
3. 點擊左上角的「未命名的專案」，重新命名為 **`GmailWithGemini`**
4. 在編輯器中，你會看到預設的 `Code.gs` 檔案
5. **刪除** `Code.gs` 中的所有預設內容
6. 開啟本專案提供的 `Gmail-with-gemini.gs` 檔案，**複製全部內容**，貼到 `Code.gs` 中
7. 點擊編輯器上方的 **「+」** 按鈕 → 選擇 **「指令碼」**
8. 將新檔案命名為 **`CHANGELOG`**（不需要加 `.gs`，系統會自動加上）
9. 開啟本專案提供的 `CHANGELOG.gs` 檔案，**複製全部內容**，貼到新建的 `CHANGELOG.gs` 中
10. 點擊 **💾 儲存**（或按 `Ctrl + S`）

### 設定 appsscript.json 資訊清單

> [!IMPORTANT]
> 這個步驟非常重要！`appsscript.json` 定義了腳本所需的 Gmail 和 Spreadsheet 權限。如果不正確設定，腳本將無法存取你的 Gmail。

1. 點擊左側的 **「專案設定」⚙️**（齒輪圖示）
2. 找到 **「在編輯器中顯示「appsscript.json」資訊清單檔案」** 選項
3. **勾選** 該選項
4. 回到 **「編輯器」**（左側 `< >` 圖示）
5. 你現在會在檔案列表中看到 `appsscript.json`，點擊開啟它
6. **刪除所有現有內容**，將本專案提供的 `appsscript.json` 內容**完整覆蓋貼入**
7. 點擊 **💾 儲存**

---

## 🔑 第三步：設定 API 金鑰

> [!NOTE]
> 將 API Key 存放在「指令碼屬性」中，而非直接寫在程式碼裡，這是安全最佳做法。即使你分享了程式碼，API Key 也不會外洩。

1. 點擊左側的 **「專案設定」⚙️**
2. 向下滑動到頁面最下方的 **「指令碼屬性」** 區塊
3. 點擊 **「新增指令碼屬性」**
4. 填入以下資訊：
   - **屬性（Property）**：`GEMINI_API_KEY`
   - **值（Value）**：貼上你在第一步取得的 API Key
5. 點擊 **「儲存指令碼屬性」**

---

## 🔍 第四步：驗證 API Key 狀態

在正式使用之前，先確認 API Key 設定正確且可用。

1. 回到 **「編輯器」**（左側 `< >` 圖示）
2. 在編輯器上方的**函式下拉選單**中，選擇 **`checkApiKeyStatus`**
3. 點擊 **「執行」▶️** 按鈕
4. **首次執行時，系統會要求授權：**
   - 點擊 **「審查權限」**
   - 選擇你的 Google 帳號
   - 如果出現「Google 尚未驗證這個應用程式」的警告畫面：
     - 點擊 **「進階」**
     - 點擊 **「前往 GmailWithGemini（不安全）」**
   - 點擊 **「允許」**
5. 查看下方的 **「執行記錄」**，確認顯示 API Key 有效的訊息

> [!TIP]
> 如果出現 API Key 無效的錯誤，請回到第三步確認你的 `GEMINI_API_KEY` 屬性值是否正確貼上，注意前後不要有多餘的空格。

---

## 🚀 第五步：手動測試執行

確認 API Key 有效後，進行第一次手動分類測試。

1. 在函式下拉選單中選擇 **`autoOrganizeGmailWithGemini`**
2. 點擊 **「執行」▶️**
3. 等待執行完成（通常需要 1–3 分鐘，取決於未讀郵件數量）
4. 查看 **「執行記錄」**，確認：
   - 系統成功讀取了郵件
   - AI 正確分析並回傳了分類結果
   - 標籤已成功套用
5. 打開你的 **Gmail**，確認：
   - 左側已出現新的分類標籤
   - 郵件已被正確歸類

> [!NOTE]
> 第一次執行時，系統會自動在你的 Google 雲端硬碟建立一個名為 `GmailWithGemini_Rules` 的試算表，用來記錄所有分類結果與執行日誌。

## ⏰ 第六步：設定每 6 小時自動執行

你可以選擇以下兩種方式之一來設定自動觸發器：

### 方法 A：使用內建函式（✅ 推薦）

這是最簡單的方式，一鍵完成設定：

1. 在函式下拉選單中選擇 **`setupTriggers`**
2. 點擊 **「執行」▶️**
3. ✅ 完成！系統會自動設定觸發器，在每天的以下時段執行：
   - 🕛 **00:00**（午夜）
   - 🕕 **06:00**（早上）
   - 🕛 **12:00**（中午）
   - 🕕 **18:00**（傍晚）

### 方法 B：手動設定觸發器

如果你想要自訂觸發頻率，可以手動設定：

1. 點擊左側的 **「觸發條件」⏰**（時鐘圖示）
2. 點擊右下角的 **「新增觸發條件」**
3. 進行以下設定：

| 設定項目           | 選擇值                           |
| :----------------- | :------------------------------- |
| 選擇要執行的函式   | `autoOrganizeGmailWithGemini`    |
| 選擇要部署的版本   | `Head`                           |
| 選擇活動來源       | **時間驅動**                     |
| 選擇以時間驅動的類型 | **小時計時器**                  |
| 選擇小時間隔       | **每 6 小時**                    |

4. 點擊 **「儲存」**

## 📊 第七步：查看分類歷史日誌

系統會自動將所有分類記錄保存到 Google 試算表中，方便你隨時查閱。

### 如何找到日誌檔案

- 前往 [Google 雲端硬碟](https://drive.google.com/)
- 搜尋 **`GmailWithGemini_Rules`**
- 開啟該試算表

### 工作表說明

| 工作表名稱          | 說明                                                       |
| :------------------ | :--------------------------------------------------------- |
| **AI_Rules**        | 每封郵件的分類記錄，包含寄件者、主旨、AI 判定的分類標籤等  |
| **AI_Execution_Log** | 每次執行的統計摘要，包含執行時間、處理封數、成功/失敗數等 |

> [!TIP]
> 你可以定期檢視 `AI_Rules` 工作表，觀察 AI 的分類是否符合你的期望。如果發現分類不準確，可以在程式碼中調整 Prompt 來優化分類效果。

## ❓ 常見問題

### Q: 免費額度夠用嗎？

**綽綽有餘！** 以下是簡單的計算：

| 項目               | 數值                             |
| :----------------- | :------------------------------- |
| 每天觸發次數       | 4 次（每 6 小時一次）            |
| 每次處理郵件數     | 最多 50 封                       |
| 每天最大 API 呼叫數 | 4 × 50 = **200 次**             |
| 免費額度           | 約 **1,500 次/天**               |
| 使用率             | 200 ÷ 1,500 = **僅 13%**        |

結論：即使每次都處理到上限 50 封郵件，使用率也僅佔免費額度的 13%，非常充裕。

---

### Q: 出現 429 錯誤怎麼辦？

**不需要擔心！** 429 錯誤表示「請求過於頻繁」。v2.0 版的腳本已內建**自動重試機制（Exponential Backoff）**，遇到 429 錯誤會自動等待後重試，不需要任何人工處理。

---

### Q: 執行紀錄出現 `Code: 403, Body: {"error": {"message": "Gmail API has not been used in project..."}}` 怎麼辦？

這是正常的安全保護機制。由於我們的腳本不僅會標籤信件，還會進一步幫你把信件移動到正確的「收件匣分頁」（例如「促銷內容」、「社交網路」），這需要呼叫進階的 Gmail API。

**解決方法非常簡單，只需 3 個步驟：**
1. 回到 Google Apps Script 編輯器畫面。
2. 在左側選單中找到 **「服務」 (Services)**，點擊旁邊的 **「+」** 號。
3. 在彈出的選單中，往下滑找到並選擇 **「Gmail API」**，然後點擊右下角的 **「新增」 (Add)**。

完成後再次執行即可，系統會自動在背景為你啟用該權限！

---

### Q: 如何停止自動執行？

有兩種方式：

- **方式一（推薦）**：在編輯器中選擇 `removeTriggers` 函式，點擊「執行」▶️
- **方式二**：前往左側「觸發條件」⏰ 頁面，手動刪除對應的觸發器

---

### Q: 如何更換 API Key？

1. 前往 **「專案設定」⚙️**
2. 滑到最下方的 **「指令碼屬性」**
3. 找到 `GEMINI_API_KEY`，點擊編輯（✏️）
4. 將值替換為新的 API Key
5. 點擊 **「儲存指令碼屬性」**

### Q: 免費版有什麼限制？

| 限制項目         | 說明                                            |
| :--------------- | :---------------------------------------------- |
| 請求頻率限制     | 每分鐘最多 10–15 次請求                         |
| 每日請求上限     | 每天最多約 1,500 次請求                         |
